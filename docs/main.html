homesick

Bibliothèque GUI pour executors Roblox — Velocity · Solara · Volt · Xeno · Delta · Fluxus · Synapse X · Electron · Krnl · Hydrogen · Arceus X et tout executor compatible UNC.
Rendu via Drawing API — aucun Instance.new, aucun ScreenGui.
API fluide par chaînage, thème sombre avec accent jaune citron.
Un layer de compatibilité automatique détecte et normalise toutes les APIs exploit au démarrage.

Chargement : collez l'URL raw du script dans loadstring(game:HttpGet(...))(). Le retour est l'objet homesick.
lua
local hs = loadstring(game:HttpGet(
    "https://raw.githubusercontent.com/FRITExCN/HomeSick/refs/heads/main/homesick.lua"
))()

-- hs est maintenant disponible, ou via _G.homesick
local hs = _G.homesick

Compatibilité Exploits

La lib injecte automatiquement un compat layer au démarrage. Aucune configuration nécessaire — il détecte l'executor actif et mappe toutes les APIs critiques.

APIs normalisées automatiquement

ismouse1pressed() -- clic gauche souris
ismouse2pressed() -- clic droit souris
iskeypressed(vkId) -- touche clavier par VK code
isrbxactive() -- fenêtre Roblox au premier plan
setrobloxinput(bool) -- bloquer/débloquer l'input Roblox
keypress(vk) / keyrelease(vk) -- simulation clavier (clipboard)
writefile / readfile / makefolder / isfile / isfolder -- I/O fichiers config/thème
Drawing.new(kind) + Drawing.Fonts -- rendu Drawing API
Fallback garanti : si une API est absente sur l'executor, le compat layer fournit un équivalent via UserInputService ou une fonction no-op silencieuse. La lib ne plantera jamais sur une API manquante.
Clipboard paste (keypress/keyrelease) nécessite que l'executor expose ces fonctions. Sur les exploits qui ne les ont pas (ex: Hydrogen), le bouton coller dans les textboxes est no-op mais le reste du GUI fonctionne parfaitement.

Ordre de résolution

Pour chaque API, la lib tente dans cet ordre :

résolution interne
-- 1. Nom UNC standard dans _G  (ismouse1pressed, iskeypressed, ...)
-- 2. Alias alternatif dans _G   (isMouseButton1Down, isKeyDown, ...)
-- 3. Nom dans env (getfenv)      si disponible
-- 4. Fallback UserInputService  (toujours disponible côté Roblox)
-- 5. Stub no-op silencieux      jamais de crash

Quickstart — script minimal

Un exemple complet du minimum nécessaire pour afficher une fenêtre avec un toggle.

lua — minimum viable
local hs = loadstring(game:HttpGet(
    "https://raw.githubusercontent.com/FRITExCN/HomeSick/refs/heads/main/homesick.lua"
))()

local win = hs.createWindow("Mon Script", 480, 420)
win:setBadge("Mon Script  |  by moi")

local tab  = win:addTab("Main")
local left = tab:addSection("Options", "Left")

left:addToggle("espEnabled", "ESP", false, function(val)
    espEnabled = val
end)

win.visible = true

Fenêtre

hs.createWindow(title, width, height) retourne Window

Crée et centre la fenêtre principale. À appeler une seule fois.

ParamètreTypeDescription
titlestringTitre affiché dans la titlebar
widthnumberLargeur en pixels (défaut : 430)
heightnumberHauteur en pixels (défaut : 500)
exemple
local win = hs.createWindow("Sell Lemons", 480, 420)
win:setBadge(text) retourne win

Texte affiché dans le footer en bas de fenêtre (ex : nom du script et auteur).

exemple
win:setBadge("Sell Lemons  |  by neaxus")
win.visible — propriété

Affiche ou masque la fenêtre. Lecture/écriture.

exemple
win.visible = true   -- ouvrir
win.visible = false  -- fermer
print(win.visible)    -- true / false
win:autoloadConfig(name)  /  win:autoloadTheme(name) chaînable

Charge automatiquement un fichier config/thème depuis homesick/<name>.json avant le premier render.

exemple
win:autoloadConfig("monscript")
    :autoloadTheme("darkgold")
win:render()

Déclenche l'ouverture de la fenêtre. Appel optionnel — mettre win.visible = true suffit.

win:remove()

Détruit complètement la fenêtre et tous ses Drawing objects. Équivalent de ui:Destroy().

exemple
win:remove()

Tab

win:addTab(name) retourne Tab

Ajoute un onglet à la bande de navigation. Retourne l'objet Tab sur lequel on appelle addSection.

exemple
local tab1 = win:addTab("Main")
local tab2 = win:addTab("Config")
local tab3 = win:addTab("Visuals")

Section

tab:addSection(name, side) retourne Section

Crée une section (carte) dans la colonne choisie. C'est sur la section qu'on ajoute tous les composants.

ParamètreTypeDescription
namestringTitre de la section
side"Left" | "Right"Colonne gauche ou droite
exemple
local left  = tab:addSection("Farming",  "Left")
local right = tab:addSection("Control",  "Right")

Toggle

section:addToggle(id, label, default, callback) retourne ToggleHandle

Case à cocher animée avec état vrai/faux. Le callback est appelé à chaque changement avec la nouvelle valeur.

ParamètreTypeDescription
idstringIdentifiant unique (utilisé pour la sauvegarde config)
labelstringTexte affiché
defaultboolValeur initiale (true = activé)
callbackfunction(bool)Appelé à chaque toggle avec la nouvelle valeur
exemple basique
local togESP = left:addToggle("espEnabled", "ESP", false, function(val)
    espEnabled = val
end)

Méthodes du handle

handle:Set(bool)  -- forcer une valeur sans déclencher le callback
handle:addKeybind(key, mode, canChange, cb)  -- ajouter un raccourci clavier
handle:addColorpicker(label, color, overwrite, cb, alpha)  -- colorpicker inline
handle:DependsOn(autreHandle)  -- grise si l'autre est désactivé
exemple avec keybind
left:addToggle("autoBuy", "Auto Buy", false, function(v)
    autoBuyActive = v
end):addKeybind("1", "Toggle", true, function() end)
forcer une valeur
togESP.rawItem:Set(true)   -- active sans déclencher callback

Checkbox

section:addCheckbox(id, label, default, callback) retourne CheckboxHandle

Identique à Toggle en comportement. Mêmes méthodes disponibles (addKeybind, addColorpicker, DependsOn, Set).

exemple
right:addCheckbox("autoMini", "Auto Minigame", false, function(v)
    MG.active = v
end)
Checkbox et Toggle sont interchangeables dans homesick. Utilisez celui qui correspond sémantiquement à votre cas.

Button

section:addButton(label, callback) retourne ButtonHandle

Bouton cliquable. Le callback est appelé sans argument à chaque clic.

ParamètreTypeDescription
labelstringTexte du bouton
callbackfunction()Appelé au clic
exemple
right:addButton("Stop All", function()
    autoBuyActive   = false
    lemonFarmActive = false
    print("Tout stoppé")
end):addKeybind("5", "Toggle", true, function() end)
-- Note : addKeybind sur un Button passe via l'API Toggle interne

Slider

section:addSlider(id, label, default, step, min, max, suffix, callback) retourne SliderHandle

Glissière numérique. La valeur est snap sur step et clampée entre min et max.

ParamètreTypeDescription
idstringIdentifiant unique
labelstringTexte affiché
defaultnumberValeur initiale
stepnumberPas de la valeur (ex : 1, 0.1)
minnumberValeur minimale
maxnumberValeur maximale
suffixstringUnité affichée (ex : "%", " ms")
callbackfunction(number)Appelé à chaque changement avec la valeur
exemple
left:addSlider("fovSize", "FOV", 90, 1, 10, 180, "°", function(val)
    fovSize = val
end)

-- Pas décimal
left:addSlider("walkSpeed", "WalkSpeed", 16, 1, 0, 100, "", function(v)
    game.Players.LocalPlayer.Character.Humanoid.WalkSpeed = v
end)

Dropdown

section:addDropdown(id, label, choices, default, callback) retourne DropdownHandle

Liste déroulante à sélection unique. Le callback reçoit un tableau avec la valeur choisie.

ParamètreTypeDescription
idstringIdentifiant unique
labelstringTexte affiché au-dessus
choicestableListe des options disponibles
defaulttable | stringValeur(s) par défaut — passer un tableau
callbackfunction(table)Appelé avec le tableau des valeurs sélectionnées

Méthodes du handle

handle:Set({valeur})            -- forcer une sélection
handle:updateChoices(newList)   -- remplacer la liste
handle:addChoice(choice)        -- ajouter une option
handle:removeChoice(choice)    -- supprimer une option
handle:clearChoices()           -- vider la liste
handle:setSearchable(bool)     -- barre de recherche intégrée
handle:setMaxSelections(n)    -- limite sélections (multi seulement)
exemple dropdown simple
local ddTeam = left:addDropdown("team", "Équipe",
    {"Rouge", "Bleu", "Vert"},
    {"Rouge"},
    function(val)
        print("Sélection :", val[1])
    end
)

-- Ajouter dynamiquement une option
ddTeam:addChoice("Jaune")

-- Forcer la sélection
ddTeam.rawItem:Set({"Bleu"})
section:addMultiDropdown(id, label, choices, default, callback) retourne DropdownHandle

Identique à addDropdown mais autorise plusieurs sélections simultanées. Le callback reçoit un tableau de toutes les valeurs cochées.

exemple multi
left:addMultiDropdown("stends", "Stends activés",
    {"Lemon Stand", "LemonDash", "Lemon Depot"},
    {"Lemon Stand"},
    function(vals)
        for _, v in ipairs(vals) do print(v) end
    end
):setMaxSelections(3)

Textbox

section:addInput(id, label, default, callback) retourne InputHandle

Champ texte éditable. Supporte Ctrl+C/V/Z/A, répétition de touche, et Tab pour naviguer entre les textboxes.

ParamètreTypeDescription
idstringIdentifiant unique
labelstringPlaceholder affiché quand vide
defaultstringValeur initiale
callbackfunction(string)Appelé à chaque frappe
exemple
left:addInput("playerName", "Nom du joueur", "", function(v)
    targetName = v
end)

Colorpicker

section:addColorpicker(id, label, default, callback) retourne ColorHandle

Sélecteur de couleur HSV complet avec canal alpha et input hex. Clic droit pour copier/coller la couleur.

ParamètreTypeDescription
idstringIdentifiant unique
labelstringNom de la couleur
defaultColor3Couleur initiale
callbackfunction(Color3, alpha)Appelé à chaque modification
exemple
left:addColorpicker("espColor", "Couleur ESP",
    Color3.fromRGB(255, 0, 0),
    function(color, alpha)
        espColor = color
        espAlpha = alpha
    end
)

Label

section:addText(text) retourne TextHandle

Texte non interactif. Le handle expose setValue(str) pour le mettre à jour dynamiquement.

exemple
local statusLbl = right:addText("En attente...")

-- Mettre à jour plus tard :
statusLbl.rawItem:Set("Actif !")

Divider / Séparateur

section:addSeparator()

Ligne horizontale de séparation visuelle entre les composants.

exemple
right:addToggle("autoDeal", "Auto Deal", true, function(v) end)
right:addSeparator()
right:addToggle("stopAll", "Stop All", false, function(v) end)

Keybind

handle:addKeybind(key, mode, canChange, callback) chaînable retourne KeyHandle

Attache un raccourci clavier à un Toggle ou Checkbox. Affiche un badge [KEY] [MODE] à droite du label.

ParamètreTypeDescription
keystringTouche : "1"–"9", "f1"–"f12", "a"–"z", ou combo "ctrl+k"
mode"Toggle" | "Hold" | "Always"Toggle = bascule, Hold = actif tant que tenu
canChangeboolSi true, clic droit sur le badge pour rebinder
callbackfunction(vkId, mode)Appelé quand le bind change
exemple — touche simple
left:addToggle("esp", "ESP", false, function(v)
    espOn = v
end):addKeybind("f2", "Toggle", true, function() end)
exemple — combo touches
left:addToggle("aimbot", "Aimbot", false, function(v)
    aimbotOn = v
end):addKeybind("ctrl+e", "Hold", true, function() end)
En mode Toggle, la touche bascule la valeur à chaque pression. En mode Hold, la valeur est true seulement tant que la touche est maintenue.

Colorpicker inline sur Toggle

handle:addColorpicker(label, color, overwrite, callback, alpha) chaînable retourne ColorHandle

Attache une petite pastille colorée directement sur la ligne d'un Toggle/Checkbox. Clic sur la pastille ouvre le picker complet.

ParamètreTypeDescription
labelstringNom du picker (affiché dans le header du popup)
colorColor3Couleur initiale
overwritebool | numbertrue pour activer l'override, ou valeur d'alpha initiale
callbackfunction(Color3, alpha)Appelé à chaque changement
alphanumberAlpha initial (0–1, défaut 1)
exemple
left:addToggle("chams", "Chams", false, function(v)
    chamsOn = v
end)
    :addKeybind("c", "Toggle", true, function() end)
    :addColorpicker("Couleur Chams", Color3.fromRGB(0,200,255), false,
        function(c,a) chamsColor=c;chamsAlpha=a end
    )

DependsOn

handle:DependsOn(parentHandle) chaînable

Grise et désactive automatiquement ce composant quand le parent est false. Aucun clic n'est possible si le parent est désactivé.

exemple
local togESP  = left:addToggle("esp",  "ESP",        false, function(v) end)
local togBox  = left:addToggle("box",  "Box ESP",     true,  function(v) end)
local togName = left:addToggle("name", "Nametag ESP", true,  function(v) end)

-- Box et Nametag grisés si ESP principal est off
togBox.rawItem:DependsOn(togESP.rawItem)
togName.rawItem:DependsOn(togESP.rawItem)

Notifications

hs:Notify(title, description, duration)

Affiche une notification en bas à droite de l'écran avec une barre de progression.

ParamètreTypeDescription
titlestringEn-tête de la notification (affiché en accent)
descriptionstringCorps du message
durationnumberDurée en secondes (défaut : 5)
exemple
hs:Notify("Cash Vine", "Prête à collecter !", 4)

-- Aussi disponible via la fonction globale notify() :
notify("Prête à collecter !", "Cash Vine", 4)
Homesick intercepte aussi les appels à print() et warn() pour les afficher comme notifications automatiquement.

Watermark

ProjectState.watermarkEnabled / watermarkTitle

Petit overlay draggable affiché en dehors de la fenêtre principale, utile pour afficher un statut quand le menu est fermé. Accessible via _G.homesickUI.

exemple
local ui = _G.homesickUI

-- Activer le watermark
ui:RegisterActivity(function()
    return autoBuyActive and "AutoBuy ON" or ""
end)

Activity

ui:RegisterActivity(callback) retourne { Remove() }

Enregistre une fonction appelée chaque frame. Si elle retourne une chaîne, celle-ci est affichée dans le watermark. Utile pour des statuts temps réel.

exemple
local ui  = _G.homesickUI
local act = ui:RegisterActivity(function(ui, dt, now)
    if autoBuyActive then
        return "AutoBuy: " .. totalBought
    end
end)

-- Supprimer plus tard
act.Remove()

Config & Thème

Homesick sauvegarde automatiquement la config dans homesick/config.json via l'onglet Settings (icône ⚙ dans la titlebar). Vous pouvez aussi forcer via code.

ui:GetValue / ui:SetValue via path

Lire ou écrire la valeur d'un item par son chemin "TabName.SectionName.Label".

exemple
local ui = _G.homesickUI

-- Lire
local v = ui:GetValue("Main.Farming.Auto Buy")

-- Écrire
ui:SetValue("Main.Farming.Auto Buy", true)
ui:SetTheme(overrides)

Remplace des couleurs du thème. Clés disponibles : accent, bg, surface, surface2, surface3, text, sub, border, green, red, yellow, unsafe.

thème custom
local ui = _G.homesickUI
ui:SetTheme({
    accent   = Color3.fromRGB(255, 214,  60),  -- jaune citron
    bg       = Color3.fromRGB( 33,  29,  17),
    surface  = Color3.fromRGB( 27,  24,  14),
})

Destroy

win:remove()  /  ui:Destroy()

Détruit complètement tous les Drawing objects et arrête le RenderStepped. À appeler lors d'un re-exec ou d'un rechargement du script.

exemple — cleanup au re-exec
-- En haut de votre script, avant de charger homesick :
if _G.MatchaCleanup then
    pcall(_G.MatchaCleanup)
end

-- À la fin, enregistrer le cleanup :
_G.MatchaCleanup = function()
    win:remove()
end

Script complet — exemple

Reproduit exactement la structure du GUI de Sell Lemons v18.18.

lua — script complet
-- Cleanup re-exec
if _G.MatchaCleanup then pcall(_G.MatchaCleanup) end

-- Variables état
local autoBuyActive   = false
local lemonFarmActive = false
local autoStandActive = false
local cashFarmActive  = true
local autoDealActive  = true

-- Chargement homesick
local hs = loadstring(game:HttpGet(
    "https://raw.githubusercontent.com/FRITExCN/HomeSick/refs/heads/main/homesick.lua"
))()

-- Fenêtre
local win = hs.createWindow("Sell Lemons", 480, 420)
win:setBadge("Sell Lemons  |  by neaxus")

-- ── Onglet Main ──────────────────────────
local tab1  = win:addTab("Main")
local left  = tab1:addSection("Farming", "Left")
local right = tab1:addSection("Control", "Right")

-- Farming
left:addToggle("autoBuy", "Auto Buy", false, function(v)
    autoBuyActive = v
end):addKeybind("1", "Toggle", true, function() end)

left:addToggle("lemonFarm", "Lemon Farm", false, function(v)
    lemonFarmActive = v
end):addKeybind("2", "Toggle", true, function() end)

left:addToggle("autoStand", "Auto Stand", false, function(v)
    autoStandActive = v
end):addKeybind("3", "Toggle", true, function() end)

left:addToggle("cashFarm", "Cash Farm", true, function(v)
    cashFarmActive = v
end):addKeybind("4", "Toggle", true, function() end)

-- Control
right:addToggle("autoDeal", "Auto Deal", true, function(v)
    autoDealActive = v
end)

right:addToggle("autoMini", "Auto Minigame", false, function(v)
    MG.active = v
end)

right:addSeparator()

right:addToggle("stopAll", "Stop All", false, function(v)
    if v then
        autoBuyActive   = false
        lemonFarmActive = false
        autoStandActive = false
        cashFarmActive  = false
    end
end):addKeybind("5", "Toggle", true, function() end)

-- ── Onglet Auto ───────────────────────────
local tab2   = win:addTab("Auto")
local stands = tab2:addSection("Stands", "Left")
local minis  = tab2:addSection("Minigames", "Right")

local standEnabled = {}
for _, nm in ipairs({
    "Lemon Stand","LemonDash","Lemon Depot",
    "Lemon Labs","Lemon Trading","Lemon Robotics","Lemon Republic"
}) do
    standEnabled[nm] = true
    stands:addCheckbox("stand_"..nm, nm, true, function(v)
        standEnabled[nm] = v
    end)
end

minis:addCheckbox("mini_race", "Race", true, function(v) end)
minis:addCheckbox("mini_trade", "Trade (soon)", false, function(v) end)

-- Ouvrir
win.visible = true

-- Cleanup
_G.MatchaCleanup = function()
    win:remove()
end
